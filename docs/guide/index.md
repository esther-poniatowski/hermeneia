# User Guide

This guide provides one connected path from setup to audit.
Each page answers a distinct question, then hands off to the next decision point.

Follow this sequence:

1. Start with `installation` and `usage` to run the linter end to end.
2. Read `writing-quality-model` and `rule-registry` to understand why findings exist and what each rule checks.
3. Use `metrics` to interpret supporting readability signals without treating them as standalone verdicts.
4. Use `configuration` and `cli-reference` to tune policy and command behavior for the target document context.
5. Finish with `prose-audit-protocol` and `sources-and-further-reading` for human-audit method and external grounding.
